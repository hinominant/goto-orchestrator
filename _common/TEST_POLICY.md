# Test Policy

> テスト品質を維持するためのルール。

---

## Core Rules

### SKIP = FAIL

- テストの SKIP は「通った」ではなく **「未完了」**
- SKIP があるまま「全テスト通過」と報告しない
- SKIP の理由を把握し、解消可能なら解消する

### テスト報告

報告時は以下を明示:

```
PASSED: N件
FAILED: N件
SKIPPED: N件（理由: ...）
```

SKIPPED が 0 でない場合、「全テスト通過」とは言わない。

### テスト実行の義務

- 実装完了後は必ずテストを実行する
- テスト未実行のまま「完了」と報告しない
- CI パイプラインがある場合はその結果も確認する
