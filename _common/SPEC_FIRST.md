# Spec-First Protocol

> 「AIに『何を作るか』を任せるな。『どう作るか』を任せろ。」

---

## Principle

仕様決定は人間の責務。実装が AI の役割。この分業が最速を実現する。

## Pipeline

```
Spec（仕様）→ Test（テスト）→ Implement（実装）
```

1. **Spec**: 人間またはScribeが仕様を定義する
2. **Test**: Radarがテストケースを先に書く
3. **Implement**: Builderがテストを通す実装を行う

## When to Apply

| 状況 | 適用 |
|------|------|
| 入出力が明確に定義できるタスク | 適用する |
| テストケースが事前に書けるタスク | 適用する |
| 探索的な調査・分析 | 適用しない |
| 正解が存在しない意思決定 | 適用しない |

## Anti-Patterns

- 仕様なしでいきなり実装を始める
- テストを後から書く（実装に引きずられる）
- 「答えがない問題」にこのパターンを無理に適用する
